User Drop-off Recovery Flow

From Amplitude → CRM → automated follow-ups → human escalation

📈

Amplitude

User behavior data
  • Events
  • Funnel steps
  • Drop-off points
  • User properties
🗄️

CRM

User Profile & Drop-off Details
  • Where user dropped
  • Last active timestamp
  • Contact details (Email / WhatsApp)
  • Platform (Web / iOS / Android)
  • Location (Country / City)
  • User properties & journey data
  • De-duplicated — one record per user
👤 Single User ViewIdentity-matched (phone / email / user-id) → no duplicate contacts or double outreach
✉️💬

Automated Outreach

Send Email / WhatsApp with a personalized message
Message includes:
  • Where user dropped
  • What they were doing
  • How to continue
  • CTA to complete action
⏳

Wait
48 Hours

🛰️

Activity Guard

During the 48h wait, watch for a publish / new-activity event or a changed last-active timestamp. If the user returns → halt instantly, mark Recovered.
🎧

Notify
Customer Care Team

Task created in CRM
  • Physical follow-up
  • Preferred call
  • User details attached
  • Drop-off step & last activity included
📞

Customer Care Calls User

CRM Call Count = 1
Post call: Update Call Status
  • Connected
  • Interested
  • Not Interested
  • Callback Requested
  • No Response
  • Wrong Number
  • Other (Add Note)
Was user connected / status updated?
✅

Update Stored in CRM

  • Status stored
  • Call notes saved
  • User journey updated
End / Close LoopNo further action
📊

Reporting & Insights

  • Drop-off analysis
  • Recovery rate
  • Channel performance
  • Call outcomes
  • Team performance
⏳

Wait
48 Hours

🛰️

Activity Guard

Same check before Sales: if last-active changes or any new activity, halt — a returned user is never handed to Sales.
👥

Re-trigger Notification to Sales Team

If no update / user still inactive
  • Task created in CRM
  • Sales to connect with user
  • All user details & history attached
